# 小牛逻辑拼图自动求解器使用说明

`Catch Cattle.html` 是一个用于辅助求解“小牛逻辑拼图”的本地网页工具。打开页面后，上传棋盘截图并确认行列数，工具会按颜色区域分析棋盘，并在结果区域标出推荐的小牛放置位置。

整个过程都在浏览器本地完成，不需要安装依赖，也不会上传图片到服务器。

## 文件组成

- `Catch Cattle.html`：主程序文件，包含界面、图片上传、颜色识别和求解逻辑。
- `README.md`：使用说明文档。

## 打开方式

1. 在当前文件夹中找到 `Catch Cattle.html`。
2. 用浏览器打开该文件。
3. 页面显示“小牛逻辑拼图自动求解器”后即可开始使用。

建议使用 Chrome、Edge、Firefox 等现代浏览器，以获得更稳定的图片读取和页面显示效果。

## 基本使用流程

1. 准备棋盘截图。
   截图应尽量只包含游戏棋盘，不要包含过多背景、按钮、标题栏或其他无关内容。

2. 上传截图。
   点击页面左侧的上传区域选择图片，或将图片直接拖拽到上传区域。

3. 核对行数和列数。
   默认棋盘大小为 `10 x 10`。如果实际棋盘不是 10 行 10 列，请在“行数”和“列数”输入框中手动修改。

4. 开始求解。
   点击“一键识别并求解”，工具会将截图按行列切分成网格，并采样每个格子的中心颜色。

5. 查看结果。
   如果求解成功，右侧“求解结果”区域会生成棋盘。带有小牛图标的格子，就是工具推荐的小牛放置位置。

## 截图要求

截图质量会直接影响识别结果。为了提高成功率，建议遵循以下原则：

- 尽量只截取棋盘区域。
- 保持棋盘边缘平直，不要倾斜或透视变形。
- 避免模糊、压缩严重或分辨率过低的截图。
- 保证每个颜色区域清晰可辨。
- 如果结果异常，先尝试重新裁剪一张更干净的棋盘截图。

## 行数与列数说明

当前版本在上传图片后会提示“系统已尝试匹配网格，请核对行列数”，但最终仍以输入框中的行数和列数作为切分依据。

行列数填写错误会导致采样位置偏移，进而影响颜色识别和求解结果：

- 行数过少时，多个实际格子可能会被合并成一个采样区域。
- 行数过多时，一个实际格子可能会被拆成多个采样区域。
- 列数填写错误也会造成同样的问题。

如果提示未找到可行解，或者生成的结果明显不符合棋盘结构，请优先检查行数和列数。

## 求解规则

工具会按照以下约束寻找小牛位置：

- 每一行放置一只小牛。
- 每一列最多放置一只小牛。
- 同一个颜色区域最多放置一只小牛。
- 任意两只小牛不能相邻，包括上下、左右和斜对角相邻。

当所有行都能满足这些条件时，页面会显示求解成功，并渲染对应结果。

## 常见问题

### 上传图片后为什么没有自动显示结果？

上传后需要手动点击“一键识别并求解”。图片上传只负责读取和预览，不会立即执行求解。

### 为什么求解结果看起来不准确？

常见原因是截图没有准确裁剪到棋盘区域，或者行数、列数填写不正确。建议先确认棋盘尺寸，再重新上传一张只包含棋盘的截图。

### 为什么会提示未找到可行解？

可能有以下几种原因：

- 行数或列数填写错误。
- 截图中包含棋盘外的内容。
- 图片过于模糊，导致颜色区域识别不稳定。
- 棋盘格子存在阴影、渐变或装饰图案，影响中心点取色。
- 当前棋盘本身不存在满足规则的解。

### 图片会被上传或保存吗？

不会。工具只使用浏览器读取本地图片，并在当前页面中进行分析。关闭页面后，上传的图片和求解结果不会被工具保存。

## 当前限制

当前版本主要通过“格子中心点取色”来识别颜色区域。这种方式适合颜色边界清晰、截图裁剪准确的棋盘，但在以下情况下可能出现误判：

- 截图包含棋盘以外的区域。
- 棋盘边框、阴影或装饰元素较多。
- 格子内部存在明显渐变或复杂图案。
- 图片倾斜、拉伸或比例失真。

遇到识别不稳定时，通常重新裁剪截图并核对行列数即可改善结果。
